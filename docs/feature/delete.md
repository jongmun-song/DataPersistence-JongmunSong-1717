# Delete

## 개요

특정 시료 데이터를 안전하게 삭제하는 기능.

## 동작

- 삭제할 대상의 ID를 입력받아 [read](read.md)의 검색 기능으로 대상을 찾는다.
- 대상이 존재하지 않으면 삭제 불가를 알리고(`false` 반환) 종료한다 — 예외를 던지지 않는다.
- 대상이 존재하면 삭제 전 확인 절차를 거친다(대상 정보를 출력한 뒤 Y/N 확인). 이는 콘솔 UI 계층의 책임이며, 리포지토리는 확인이 끝난 삭제 요청만 처리한다.
- 확인 후 메모리 상의 시료 목록에서 대상을 제거한다. 이때 삭제 전 값과 위치(인덱스)를 함께 보관해둔다.
- [json-file-storage](json-file-storage.md)를 호출해 변경된 목록을 JSON 파일에 저장한다.
- 저장이 실패하면 보관해둔 값을 원래 위치에 복원하고(롤백), 예외를 상위로 전달한다.
- 성공 시 `true`를 반환한다.

## 관련 기능

- [read](read.md): 삭제 대상을 찾을 때 사용.
- [json-file-storage](json-file-storage.md): 삭제 결과를 파일에 반영.
