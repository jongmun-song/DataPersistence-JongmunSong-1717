# Create

## 개요

새로운 시료 데이터를 입력받아 JSON 파일에 저장하는 기능.

## 동작

- 콘솔에서 사용자로부터 시료 정보(이름, 평균 생산시간, 수율, 초기 재고 수량 등 필드)를 입력받는다.
- 입력값의 형식/필수 항목 여부를 검증한다. 필수 필드가 비어 있는 등 검증에 실패하면 `std::invalid_argument`를 던진다.
- 신규 시료에 고유 ID를 부여한다. ID는 리포지토리 내부에서 현재 목록의 최댓값 ID + 1로 자동 부여하며, 입력값에 ID가 포함되어 있어도 무시한다.
- 검증을 통과한 데이터를 `Sample` 구조체(`Model/Sample.h`)로 구성하여 메모리 상의 시료 목록에 추가한다.
- [json-file-storage](json-file-storage.md)를 호출해 변경된 목록을 JSON 파일에 저장한다.
- 저장이 실패하면 방금 추가한 시료를 메모리 목록에서 제거해 저장 전 상태로 롤백하고, 예외를 상위로 전달한다.
- 반환값: 생성된 `Sample`(부여된 ID 포함).

## 관련 기능

- [json-file-storage](json-file-storage.md): 저장된 결과를 파일에 반영.
- [read](read.md): 추가된 데이터를 이후 조회할 때 사용.
